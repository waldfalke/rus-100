Понял — единое, чистое, UX-ориентированное ТЗ без «технической хуерги». Ниже — компактное, консолидированное и готовое к использованию тобой как продуктовым дизайнером: списки экранов, поведение, состояния, элементы интерфейса, тексты/микрокопия и критерии приёма.

# Общее / предпосылки

- Аккаунт обычно имеет 5–10 **актуальных** групп; могут быть пары или несколько десятков.
    
- Есть правило архивации: каждый год прошлогодние группы отправляются в архив. Архивные скрываются по умолчанию, но есть кнопка «Показать архив».
    
- Возможность переключения аккаунтов (combo-box) — если у пользователя доступ к нескольким аккаунтам.
    
- Целевая аудитория: преподаватели (в основном зрелые, ~50 лет) — читаемость, простая навигация, явные подписи, минимум неожиданного поведения.

# 1. Dashboard — экран карточек групп (главная)

**Цель:** быстрый обзор всех актуальных групп и переход к детальной работе с одной группой.

## Структура

- Хедер: название аккаунта + combo-box выбора аккаунта (если доступно) + кнопка «Создать группу» + кнопка «Показать архив» (тумблер/фильтр).
    
- Основная зона: сетка карточек групп. Адаптация: desktop — сетка 2–4 колонки в зависимости от ширины; mobile — список карточек.
    

## Карточка группы — обязательные элементы

- Название группы (кликабельно — открывает страницу группы).
    
- Краткая метрика строкой под названием: «X учеников · Y папок тестов отображается» (не перегружаем цифрами).
    
- Быстрые действия (икон-кнопки справа вверху карточки): ⚙️ Настройки группы, ✏️ Редактировать, ⋯ Ещё (если нужно). Шестерёнка видна и доступна без прокрутки.
    
- Кнопка «Открыть группу» / «Перейти» (primary).
    
- Небольшой тултип/иконка с пояснением о видах статистики (на карточке — короткая подсказка «Статистика по группе / по заданиям / по тестам» — при ховере раскрывается объяснение).
    
- Состояния карточки: активная, неактивная (архив — стилизованная как «затенённая»), выбранная.
    

## Поведение

- Нажатие на карточку → страница группы.
    
- «Показать архив» — переключатель, который разворачивает/скрывает архивные карточки внизу списка (архивные всегда отделены визуально и могут иметь пометку «архив, год XXXX»).
    
- При большом количестве групп — поиск + сортировка (алфавит, последние активности).
    

## Дизайн-заметки

- Контрастность и крупный шрифт. Чёткие подписи. Минимум иконографии, но понятные пиктограммы рядом с действием.
    
- Карточки не содержат детальной статистики — только привлёкшие метрики и путь к деталям.

# 2. Страница группы (детальная) — single group view

**Цель:** предоставить учителю полный рабочий набор табов для анализа и управления группой.

## Верхняя шапка

- Название группы, быстрые действия: Редактировать группу, Настройки (шестерёнка), Копировать ссылку на группу, Кнопка «Архивировать/Разархивировать» (если права позволяют).
    

## Основная структура — табы/секции

Табы: **Обзор** / **Статистика по группе** / **Статистика по заданиям** / **Тесты (папки)** / **Ученики** / **Результаты** / **Лента ответов**

> Каждый таб — отдельная визуальная секция, открываемая по клику. По умолчанию открывается **Обзор**.

### Обзор

- Краткие агрегаты: число учеников, % выполнения заданий за период (выбор периода сверху), топ-5 проблемных номеров (по частоте ошибок), последние активности (ссылка в ленту ответов).
    
- CTA: «Открыть статистику по заданиям» / «Открыть статистику по папкам».
    

### Статистика по группе (агрегат)

- Плитки/блоки и компактный график: агрегированная статистика по номерам заданий и правилам (без разбивки по ученикам).
    
- Таблица/список: номер задания → кол-во ошибок / % правильных / основные правило-ошибки.
    
- Фильтр периода (день/неделя/месяц/пользовательский интервал).
    
- Экспорт (CSV/Excel) — опционально покажи кнопку, если потребуется.
    

### Статистика по заданиям (табличный вид)

- Таблица: строки — ученики; столбцы — категории/группы заданий (столбцы сгруппированы по типам).
    
- Каждая ячейка: краткое состояние (значок/процент/кол-во попыток) + кликабельная ссылка на **все результаты по этому заданию у этого ученика**.
    
- По ховеру — всплывашка с краткой историей (последние попытки).
    
- Фильтры: по категории заданий, по правилу, по ученику (поиск по имени).
    

### Тесты (папки)

- Список папок, каждая раскрывается accordion’ом: внутри — список тестов.
    
- Для выбранных папок отображается статистика (строки — ученики, столбцы — тесты).
    
- Настройка отображаемых папок: в настройках группы — чек-боксы, какие папки показывать для этой группы (teacher selected).
    
- Для каждого теста — кнопка «Открыть статистику по тесту» (дрилл-ин).
    

### Статистика по тесту (дрилл-ин)

- Если тест в формате ЕГЭ — вопросы группируются по категориям (категория → вопросы).
    
- Таблица: строки — ученики; столбцы — вопросы (или группы вопросов); в ячейках — результат по вопросу (прав/неправ, время, попытки).
    
- Фильтры: только неверные / только без попыток / сортировка по сумме очков.
    

### Ученики

- Карточки учеников (или компактный список с аватаром/инициалами): имя, последние активности, % выполнения за выбранный период, быстрый доступ к профилю ученика и его результатам.
    
- По клику — детальный просмотр ученика (всплывающее модальное окно или отдельная страница) с историей результатов и рекомендованными действиями для учителя.
    

### Результаты (вкладка, см. отдельный раздел ниже)

- Краткая навигация к полному списку результатов (фильтры/поиск).
    

### Лента ответов (вкладка, см. раздел 3)

- Краткий поток свежих действий по группе.
    

## Поведение и навигация

- В каждом табе — явные bread crumbs и CTA «Вернуться к Dashboard».
    
- Везде, где есть «статистика», должна быть явная подпись/лейбл, какой это тип статистики (чтобы убрать путаницу): «Статистика по группе (агрегат)», «Статистика по заданиям (по ученикам)», «Статистика по тестам (папки/фиксированные тесты)».

# 3. Страница «Результаты»

**Цель:** быстро найти, просмотреть или продолжить конкретный результат (тест, отработка и т. п.).

## Главные требования к UX

- Основная цель пользователя — найти конкретный результат и открыть/продолжить его. Интерфейс проектируется под эту задачу.
    
- На странице должны быть **поисковая строка** и **фильтры** (видны сверху):
    
    - Поиск по названию папки, названию теста, имени ученика, номеру задания, дате.
        
    - Фильтры: завершён/незавершён, по группе, по папке, по учителю, период (preset и пользовательский).
        
    - Сортировка: по дате завершения (по умолчанию — свежие сверху), по имени ученика, по папке.
        

## Список результатов — представление

- Иерархическая/разворачиваемая списоковая разметка: Group → Folder → Test/Assignment → Results. Каждый уровень можно свернуть/развернуть.
    
- Для незавершённых результатов видно состояние и кнопка «Продолжить».
    
- Каждая запись содержит: название, ученик, дата создания/завершения, статус (завершён/незавершён), краткая статистика (очки или %), кнопки «Открыть» / «Поделиться» (копировать ссылку).
    

## Ссылки на результаты

- Ссылка на результат стабильно работает вне зависимости от авторизации — показать рядом «иконка ссылки с подсказкой: доступно по ссылке». Микрокопия: «Ссылка ведёт напрямую к результату — можно прислать ученику или родителям».
    

## Поведение поиска/фильтров

- Поиск — live (по мере ввода) + Enter для полнотекстного поиска.
    
- Комбинация фильтров — применять вместе (например: незавершённые за последнюю неделю в группе X).
    
- Пустое состояние (ничего не найдено) — объясняющий текст и CTA «Сбросить фильтры» / «Показать последние результаты».

# 4. Лента ответов (Answers feed)

**Цель:** быстро видеть, кто и что сделал за указанный период; помогать при живой проверке классов и на паре.

## Основные возможности

- Выбор группы (если мы в группе — по умолчанию фильтр стоит на текущей группе).
    
- Выбор периода: готовые пресеты (за 1 день, за 1 неделю, за 1 месяц) + произвольный интервал.
    
- Пара вариантов группировки (переключатель):
    
    1. По тесту (каждый блок = тест; внутри — список учеников, кто сделал/кто не сделал).
        
    2. По ученику (каждый блок = карточка ученика; внутри — список его результатов за период).
        
    3. По заданию/отработке (каждый блок = задание; внутри — кто и как сделал).
        
- В карточке блока: краткая статистика (кол-во сделавших / не сделавших), дата последней активности, кнопка «Открыть список».
    

## Карточки учеников в ленте

- По умолчанию компактная карточка: аватар/инициалы, имя, мини-статистика (№ результатов в периоде), цветовая полоса/фоновая метка, показывающая степень активности/качества.
    
- Возможные цвета (прототипное правило):
    
    - Зеленый — высокий уровень выполнения/качества (порог, например, ≥80%).
        
    - Жёлтый — средний (50–79%).
        
    - Красный — низкий (<50%).
        
    - Белый/серый — отсутствуют результаты/ничего не сделал.
        
- Альтернатива: шкала относительного ранжирования — цвет зависит от отношения к максимальному количеству/результату в этой группе за период (дифференцируем по плотности активности).
    

## Визуальная дифференциация

- Цветовая полоса + маленький показатель (число результатов и средний %).
    
- Иконки: ✅ — завершил; ⚠️ — пробелы/ошибки; — — ничего не делал.
    

## Взаимодействие

- По клику на карточку ученика — раскрывается подробный список результатов за период + ссылки на каждый результат.
    
- Кнопка «Пометить как просмотрено» / «Пометить прочитанным» (опционально) — сохраняет состояние и помогает учителю отслеживать, что он уже просмотрел.
    
- Сортировка внутри вида: по активности (кол-во результатов) или по алфавиту.
    

## Пустые/крайние состояния

- Если никто ничего не делал — показывать набор рекомендаций/быстрых CTA: «Отправить напоминание» (если такая функция в будущем), «Провести тест сейчас», «Сменить период».

# Микрокопия и тексты (рекомендации)

- Карточка группы: подзаголовок «X учеников · Y папок отображается»
    
- Типы статистики (везде рядом с заголовком):
    
    - «Статистика по группе — агрегат по всем ученикам, показывает проблемные номера и правила.»
        
    - «Статистика по заданиям — разбивка по ученикам и категориям заданий.»
        
    - «Статистика по тестам — строки — ученики, столбцы — тесты; разворачиваемая по папкам.»
        
- Результаты: кнопка «Продолжить» — подсказка «Вы можете продолжить незавершённый результат в этом браузере.»
    
- Лента: периодные кнопки «За 1 день / За 1 неделю / За месяц / Пользовательский период».
    
- Архив: «Архивные группы скрыты. Показать архив» (тумблер).

# Доступные состояния UI / ошибки

- Загрузка: skeletons для карточек и таблиц.
    
- Пустой список: информативное пустое состояние + CTA (создать группу / импортировать / показать архив).
    
- Ошибка при загрузке: короткий текст ошибки + кнопка «Повторить».
    
- Много элементов: пагинация или «инфинити с кнопкой показать ещё» — для списков результатов и ленты.
    

---

# Адаптивность и доступность

- Контрастность WCAG AA минимум; крупные шрифты по умолчанию (возможность переключить размер UI — опция).
    
- Элементы управления большие и с достаточными отступами для пожилых пользователей.
    
- Таблицы: при мобильном — свёртывание групп колонок в accordion или горизонтальный скролл с фиксированной левой колонкой (имя ученика).
    
- Все иконки имеют подписи/alt-текст.

# Acceptance criteria — что должно быть проверено вручную

1. **Dashboard:** отображаются карточки групп с названием, краткой метрикой и видимыми настройками; архив скрыт по умолчанию, переключатель «Показать архив» показывает архивные карточки.
    
2. **Страница группы:** доступны табы — Обзор, Статистика по группе, Статистика по заданиям, Тесты (папки), Ученики, Результаты, Лента ответов; в каждом табе присутствует явный заголовок поясняющий тип статистики.
    
3. **Статистика по заданиям:** таблица строки=ученики, столбцы=категории/задания; каждая ячейка кликабельна и приводит к списку результатов по этому заданию/ученику.
    
4. **Тесты/папки:** папки разворачиваются, внутри — тесты; можно выбрать, какие папки показывать в группе (UI для выбора).
    
5. **Результаты:** есть поиск и фильтры; можно найти результат по названию/дате/ученику; незавершённые результаты имеют кнопку «Продолжить». Ссылка на результат видна и помечена как «можно поделиться».
    
6. **Лента:** можно выбрать период; есть переключатель группировки; карточки студентов показывают количество результатов и цветовую индикацию активности/качества.
    
7. **Адаптив:** основные экраны корректно читаются и работают на мобильном; таблицы доступны через сворачивание/скролл.

# Спецификация компонентов для Figma — Админка сервиса подготовки к ЕГЭ

Этот документ — готовая спецификация компонентной библиотеки, собранная под задачу: Dashboard (карточки групп), Таблицы (статистика/результаты), Фильтровая панель и Лента ответов (Feed). Предназначено для быстрой верстки в Figma, передачи фронтенду и использования в прототипировании.

## 3. Технические требования (наш стек)

### 3.1. Стек технологий (текущий)
- **Frontend**: React 18+ с TypeScript
- **Стилизация**: Tailwind CSS + shadcn/ui
- **Компоненты**: Radix UI (через shadcn/ui)
- **Иконки**: Lucide React
- **Анимации**: Встроенные в Radix UI + CSS transitions
- **Формы**: React Hook Form + Zod (уже настроено)
- **Состояние**: React Context + useState/useReducer
- **Дизайн-токены**: CSS Custom Properties + Tailwind

### 3.2. Структура проекта (существующая)
```
src/
├── components/
│   ├── ui/           # shadcn/ui компоненты (50+ готовых)
│   ├── feature/      # Бизнес-логика компоненты
│   └── stories/      # Storybook истории
├── pages/            # Next.js страницы
├── hooks/            # Кастомные хуки
├── lib/              # Утилиты и конфигурация
├── types/            # TypeScript типы
└── styles/           # Глобальные стили + токены
```

### 3.3. Компонентная архитектура (shadcn/ui подход)
- **Композиция**: Использование Radix UI primitives
- **Кастомизация**: CSS переменные + Tailwind классы
- **Типизация**: Полная TypeScript поддержка
- **Переиспользование**: Компоненты как building blocks
- **Консистентность**: Единая дизайн-система

### 3.4. Интеграция с существующими компонентами

#### Готовые компоненты для использования:
- `@/components/ui/card` - для карточек групп и учеников
- `@/components/ui/table` - для статистических таблиц
- `@/components/ui/form` - для редактирования групп

**Простота и ясность**

- Минимализм в дизайне, фокус на контенте
- Четкая иерархия информации через наши дизайн-токены
- Интуитивная навигация с использованием shadcn/ui паттернов

- Единообразие через shadcn/ui компоненты
- Последовательность в использовании дизайн-токенов
- Стандартизированные отступы через Tailwind spacing

**Отзывчивость**

- Mobile-first подход с Tailwind responsive классами
- Быстрая загрузка через React оптимизации
- Полная доступность через Radix UI primitives

**Целевая аудитория**

- Мобильная первая читабельность: базовые размеры и отступы рассчитаны на пользователей ~50 лет (открытый интерфейс, крупные кликабельные зоны)

- Компоненты сделаны с Auto Layout и вариациями (variants) для состояний. Имя компонента в Figma: `Component / <role> / <name> / <variant>`

## Tokens (цвета, типографика, тени, радиусы)

**Цвета (на базе shadcn/ui токенов)**

#### Основная палитра
- **Primary**: `hsl(var(--primary))` - основной бренд цвет
- **Primary Foreground**: `hsl(var(--primary-foreground))`
- **Secondary**: `hsl(var(--secondary))` - вторичный цвет
- **Secondary Foreground**: `hsl(var(--secondary-foreground))`

#### Семантические цвета
- **Success**: `hsl(142.1 76.2% 36.3%)` - зеленый для успеха
- **Warning**: `hsl(32.1 94.6% 43.7%)` - оранжевый для предупреждений  
- **Destructive**: `hsl(var(--destructive))` - красный для ошибок
- **Info**: `hsl(221.2 83.2% 53.3%)` - синий для информации

#### Нейтральные цвета
- **Background**: `hsl(var(--background))`
- **Foreground**: `hsl(var(--foreground))`
- **Muted**: `hsl(var(--muted))`
- **Muted Foreground**: `hsl(var(--muted-foreground))`
- **Accent**: `hsl(var(--accent))`
- **Border**: `hsl(var(--border))`
    

**Typography (Tailwind + наши токены)**

#### Шрифты
- **Sans**: `Inter, ui-sans-serif, system-ui` (основной)
- **Mono**: `JetBrains Mono, ui-monospace` (код)

#### Размеры (Tailwind классы)
- **H1**: `text-3xl font-bold` (30px, 700)
- **H2**: `text-xl font-semibold` (20px, 600)
- **Body**: `text-base` (16px, 400)
- **Small**: `text-sm` (14px, 400)
- **Caption**: `text-xs` (12px, 400)
    

**Elevation / shadows**

- none
    
- sm: `0 1px 2px rgba(2,6,23,0.06)`
    
- md: `0 4px 12px rgba(2,6,23,0.08)`
    

**Border radius (наши токены)**

- **Small**: `calc(var(--radius) - 2px)`
- **Medium**: `var(--radius)` (обычно 6px)
- **Large**: `calc(var(--radius) + 4px)`
- **Full**: `9999px`
